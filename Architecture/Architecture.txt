reproduire le comportement d'une vraie API.

┌────────────────────────────────────────────────────────────────────────────┐
│ Service              │ Create │ Read │ Update │ Delete │ Status             │
├──────────────────────┼────────┼──────┼────────┼────────┼────────────────────┤
│ StudentService       │   ✅   │  ✅  │   ✅   │   ✅   │ Complet (mocké)    │
│ ProgramService       │   ✅   │  ✅  │   ✅   │   ❌   │ Partiel (mocké)    │
│ SessionService       │   ✅   │  ✅  │   ✅   │   ❌   │ Partiel (mocké)    │
│ GradeService         │   ✅   │  ✅  │   ❌   │   ❌   │ Lecture + Create   │
│ AttendanceService    │   ✅   │  ✅  │   ❌   │   ❌   │ Lecture + Create   │
│ MLService            │ ✅(*)  │  ✅  │ ✅(**) │   ❌   │ Partiel (mocké)    │
│ PredictionService    │   ❌   │  ✅  │   ❌   │   ❌   │ Lecture seule      │

📊 ENTITÉS DE DONNÉES (INTERFACES TYPESCRIPT)
================================================================================

Voici les interfaces TypeScript qui définissent le modèle de données :

1. Student (Étudiant)
   ├── id: string
   ├── matricule: string
   ├── firstName: string
   ├── lastName: string
   ├── email: string
   ├── phone?: string
   ├── dateOfBirth: string
   ├── programId: string
   ├── programName: string
   ├── sessionId: string
   ├── sessionName: string
   ├── status: 'active' | 'inactive' | 'graduated'
   ├── riskLevel?: 'low' | 'medium' | 'high'
   └── riskScore?: number (0-100)

2. Program (Filière)
   ├── id: string
   ├── name: string
   ├── code: string
   ├── description?: string
   ├── duration: number (années)
   ├── studentCount: number
   └── status: 'active' | 'inactive'

3. Session (Session Académique)
   ├── id: string
   ├── name: string
   ├── year: string
   ├── startDate: string
   ├── endDate: string
   ├── status: 'active' | 'inactive' | 'completed'
   └── studentCount: number

4. Grade (Note)
   ├── id: string
   ├── studentId: string
   ├── studentName: string
   ├── subjectId: string
   ├── subjectName: string
   ├── sessionId: string
   ├── sessionName: string
   ├── value: number
   ├── maxValue: number
   ├── type: 'exam' | 'assignment' | 'project' | 'participation'
   └── date: string

5. Attendance (Présence/Absence)
   ├── id: string
   ├── studentId: string
   ├── studentName: string
   ├── date: string
   ├── status: 'present' | 'absent' | 'late' | 'excused'
   ├── subjectId: string
   ├── subjectName: string
   └── justification?: string

6. MLModel (Modèle ML)
   ├── id: string
   ├── name: string
   ├── version: string
   ├── status: 'active' | 'inactive' | 'training'
   ├── accuracy: number (0-100)
   ├── precision: number (0-100)
   ├── recall: number (0-100)
   ├── f1Score: number (0-100)
   ├── trainedAt: string
   └── trainingDataSize: number

7. Prediction (Prédiction ML)
   ├── id: string
   ├── studentId: string
   ├── studentName: string
   ├── riskScore: number (0-100)
   ├── riskLevel: 'low' | 'medium' | 'high' | 'critical'
   ├── predictedSuccessRate: number (0-100)
   ├── factors: Array<{ name: string; impact: number }>
   ├── createdAt: string
   └── modelVersion: string

8. Alert (Alerte)
   ├── id: string
   ├── studentId: string
   ├── studentName: string
   ├── type: 'performance' | 'attendance' | 'risk' | 'prediction'
   ├── level: 'low' | 'medium' | 'high' | 'critical'
   ├── message: string
   ├── status: 'new' | 'acknowledged' | 'resolved'
   ├── createdAt: string
   └── programName?: string

9. User (Utilisateur)
   ├── id: string
   ├── email: string
   ├── firstName: string
   ├── lastName: string
   └── role: 'admin' | 'teacher' | 'ds' | 'pedagogical'
